Week 4: Learning CSS styling

Publised on November 7, 2023

This week I started formatting my own CSS stylesheet for my webpage, this included using the normalize.css and stylesheet.css templates and adapting them to suit the style I was going for.

I researched into the differences between Normalize CSS and CSS Reset. From this I have learnt that Normalize CSS allows for cross-browser uniformity when styling HTML. Chrome, Safari and Firefox use the same styles. However, Internet Explorer and Edge use different styles. This means that normalize.css conforms to the IE/Edge styling and apply this to the rest of the browsers.

Reset.css takes a different approach to how these styles are applied, this will reset the styles in all browsers, and takes the styles from the project itself. CSS Resets are big chains of code that are difficult to read while debugging.

I have learnt the differences in the types of font that can be used when using CSS (Duckett, 2011). My main findings from this were that sans-serif typefaces are easier to read if the size of the text is smaller, this will be something that I will implement into my webpage. According to this book, the default size of text in web browsers is 16px, upon learning this, I have decided to use this type size as the standard for my webpage, I will be making my body text this size, then scaling up for headings.
